# 🧠 OOP என்றால் என்ன?

👉 OOP = **Object-ஐ மையமாக வைத்து programming செய்வது**

💡 Simple-ஆ:

👉 Normal programming:
**Code → Data-ஐ use பண்ணும்**

👉 OOP:
**Data + Code சேர்ந்து Object ஆகும்**

---

# 🔄 Programming evolution (easy story)

👉 பழைய காலம்:

1. Machine code → 0/1
2. Assembly language
3. High-level language (FORTRAN)
4. Structured programming (C)
5. 👉 **OOP (Java)**

## ⚠️ Problem என்ன?

👉 Programs பெரியதாக ஆனதும்:

* Bug அதிகம்

## 💡 Solution = OOP

👉 OOP என்ன பண்ணுது?

✔️ Complex problem-ஐ small objects-ஆ break பண்ணும்
✔️ Real-world மாதிரி design

# 🧠 Core idea (மிக முக்கியம்)

## 🔴 Old method:

👉 **Code acting on data**

---

## 🟢 OOP:

👉 **Data controls code**

💡 Meaning:

* Data (object) தான் decide பண்ணும்
* என்ன operations செய்யலாம்

---

# 🧱 Real-life example

👉 Car 🚗

* Data:

  * speed
  * fuel

* Code:

  * accelerate()
  * brake()

👉 இவை சேர்ந்து:
👉 **Car object**

---

# 🧩 OOP 3 pillars (மிக முக்கியம்)

👉 எல்லா OOP language-லும் இருக்கும்:

---

## 🔹 1. Encapsulation 🔒

👉 Data + methods bundle

👉 Outside world direct access முடியாது

💡 Example:

* Bank account
* balance direct access ❌
* withdraw() ✔️

---

## 🔹 2. Inheritance 🧬

👉 One class → மற்ற class inherit

💡 Example:

* Vehicle
  ↓
* Car

👉 Code reuse ✔️

---

## 🔹 3. Polymorphism 🎭

👉 Same method → different behavior

💡 Example:

```java
id="p9x6dx"
draw()
```

* Circle → draw circle
* Rectangle → draw rectangle

---
