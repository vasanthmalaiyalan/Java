சூப்பர் 🔥 இது Java-வின் **release system change** — interview-க்கும், real-world-க்கும் ரொம்ப முக்கியம். இதை super simple-ஆ breakdown பண்ணுறேன் 👇

---

# 📘 A Faster Release Schedule (Simple Tamil)

## 🕰️ பழைய system (Old Java)

👉 முன்னாடி:

* New Java version → 2–3 வருடத்துக்கு ஒருமுறை
* Features வர → wait அதிகம்

---

## ⚡ புதிய system (Modern Java)

👉 இப்போ:

✔️ **6 மாதத்துக்கு ஒருமுறை new version**

👉 இதை சொல்வது:
👉 **Feature Release**

---

## 📅 Release timing

👉 Every year:

* March
* September

👉 Example:

* JDK 10 → March
* JDK 11 → September

---

# 🚀 ஏன் இது change பண்ணினாங்க?

👉 காரணம்:

* Technology fast-ஆ change ஆகுது
* Developers new features சீக்கிரம் வேண்டும்

👉 அதனால்:

✔️ Fast updates
✔️ Quick improvements

---

# 🧠 Feature Release என்ன?

👉 ஒவ்வொரு 6 மாதமும்:

* Ready ஆன features மட்டும் release பண்ணுவாங்க

👉 Half-ready feature:
❌ wait செய்யும்

---

# ⭐ LTS (Long-Term Support)

👉 இது மிக முக்கியம் 💥

## 🔹 LTS என்றால் என்ன?

👉 Long-term support version

✔️ Long time support (years)
✔️ Stable version

---

## 📅 LTS pattern

👉 Every **3 years**:

* ஒரு LTS release

---

## 🔥 Important LTS versions

* JDK 11
* JDK 17
* (Future: 21, 25…)

---

## 🎯 ஏன் LTS important?

👉 Companies:

✔️ LTS தான் use பண்ணுவாங்க

👉 ஏன்?

* Stable
* Bugs குறைவு
* Long support

---

# ⚖️ Feature vs LTS

| Feature Release | LTS            |
| --------------- | -------------- |
| 6 மாதம்         | 3 வருடம்       |
| Short support   | Long support   |
| New features    | Stable version |

---

# 🧠 Real-world usage

👉 Developers:

* New features try → Feature release

👉 Companies:

* Production apps → LTS

---

# 🔄 Example timeline

```text id="h01kgy"
JDK 9 → 10 → 11 (LTS) → 12 → 13 → ... → 17 (LTS)
```

# 🔚 Final clarity

💡 Simple-ஆ:

👉 Java:

* Fast-ஆ update ஆகுது
* Stable version separate-ஆ இருக்கு

**“Java releases fast, but stability comes from LTS versions”**
